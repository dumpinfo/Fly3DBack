Main Frontend: flyFrontend.exe

flyFrontend.exe is the main frontend application used in the engine. It is composed by a single window where the simulation images are rendered, and a menu with all the options available in Fly3D. It can switch between windowed and fullscreen modes, and serves for viewing demos and final versions of games/apps in action.

This frontend works with Fly3D scene files (.fly), which contain all information needed to start running a given level. Upon opening an existing .fly file, the frontend calls the engine's level initialization methods, which initializes all plugin DLLs needed in the level. The BSP is then loaded, and the objects and entities are created and initialized, and the simulation begins.

The following sections deal with all the options available in flyFrontend.exe menus.

The Window Menu

flyFrontend.exe features a simple window menu, with basic options, like loading a .fly file, quitting the application and changing a great number of rendering options, which are also available through the functinos keys on the keyboard.

The File submenu has only 4 items: Open, for opening a scene file (*.fly); Close, for closing the currently open file and leaving the screen blank (like in the figure above); About, for information concerning the application developers; and Exit, for quitting flyFrontend.exe.

The View submenu has 14 items:

  • Toggle Full-Screen (Alt+Enter) does what its caption implies: switches between windowed and fullscreen modes;
  • Menu (F1) loads a special level called menu.fly, which is actually a menu level for browsing the other .fly files present in the system, creating a multiplayer game starting the server, and playing pre-recorded demos;
  • Mute (F2) turns off all sound in the simulation;
  • Save Panorama (F3) saves a panoramic screenshot centered at the current camera position, to be viewed with Paralelo's Pan3D Control;
  • Save Screen (F4) saves a screenshot in Targa format, and places it in the 'data' folder inside Fly3D installation;
  • Node Only (F5) tells the engine to draw only the BSP node(s) which the camera is inside or clips (for debugging purposes);
  • PVS (F6) toggles PVS culling on/off;
  • Mipmap (F7) toggles Mip-mapping on/off;
  • Filtering (F8) toggles texture filtering on/off;
  • Multitexture (F9) toggles Multexturing on/off;
  • Stencil (F10) toggles the use of stencil buffer for dynamic shadows on/off;
  • Wireframe (F11) toggles the wireframe mode on/off;
  • Clear Background (F12) toggles filling the background in black on/off (useful together with the wireframe mode);
  • Status (Tab) toggles drawing the 2D layer and text layer on/off.

The In-Game Menu

flyFrontend.exe also has a cool in-game menu with several options, including some of the ones found in the window menu. It is called by pressing the 'Esc' key. The in-game menu consists of four sub-menus: File, Render, Sound and About.

The File submenu includes the following options: Close, which closes the currently loaded .fly file; Menu, which loads the menu.fly level, consisting of a room where it is possible to browse for available .fly files, create and join a multiplayer game, and run a pre-recorded demo; and Exit, which quits flyFrontend.exe.

The Render submenu includes some items also available in the window menu. Refer to the previous section for explanations on the Multitexture, Texture Filter, Texture Mip-Map, Stencil Shadows, Clear Background , Draw Wireframe, Draw No PVS and Draw BSP CamNode Only options. The additional options are Dynamic Lighting, which turns all dynamic light sources on/off, and Draw Portals and Mirrors, which turns the drawing of portals and mirrors on/off.

The Sound submenu includes the Effects Volume item, which shows the sound volume for special effects, and the Mute Game Sounds item, which turns off all sounds in the simulation (just like the Mute option of flyFrontend window menu.

Finally, the About submenu shows 3 labels which are not actually menu items; they just display, respectively, the current Fly3D version that is running, the currently loaded .fly file, and the currently loaded .bsp file.

The Console

flyFrontend.exe also features a console for entering commands, that can be called by pressing the '`' key. Almost all operations with plugins and objects, like loading a DLL or activating an object, can be performed through console commands.

Below is an enumeration of all the available commands, with their correct sintax and a brief explanation of each of them:

  • get <varname> | get <objname.objparam>: queries the value of a global variable or an object parameter and shows it in the console screen;
  • set <varname> <value> | set <objname.objparam> <value>: sets the value of a global variable or object parameter to the given value;
  • map <path/level>.fly | map none: loads the given map or unloads the current map (map none);
  • save: saves the currently loaded .fly file with any modifications made during the simulation;
  • listgames | listgames <ipaddress>: lists all the multiplayer games currently online in the local network (listgames) or on the given machine (listgames <ipaddress>);'
  • connect | connect <ipaddress> | connect <ipaddress> <gamenum>: joins the first multiplayer game in the local network (connect) or on the given machine (connect <ipaddress>), or the given multiplayer game (connect <ipaddress> <gamenum>); the game number can be queried with the listgames command;
  • disconnect: disconnects from the multiplayer game to which the client is currently connected;
  • list plugins | list classes | list <classname> | list <objname>: queries the list of all plugins (list plugins) or classes(list classes), the list of instances of a given class (list <classname>), or the list of parameter and their values from a given object (list <objname>), and shows it on the console screen;
  • activate <objname>: clones the given instance of an object and inserts it into the simulation;
  • destroy <objname>: destroys the given active object, removing it from the simulation;
  • insert <plugin>.dll: loads the given plugin DLL and all of its classes;
  • remove <plugin>.dll: unloads the given plugin DLL and all of its classes;
  • playdemo <demofile>.fdm: plays the given pre-recorded demo file;
  • timedemo <demofile>.fdm: plays the given pre-recorded demo file as a time demo, without the time delay of messages;
  • exec <commandline>: executes a generic Windows command;
  • quit: exits flyFrontend.exe.
