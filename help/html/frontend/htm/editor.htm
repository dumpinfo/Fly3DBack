Parameter Editor: flyEditor.exe

As a software development kit, Fly3D 2.0 offers frontends and utilities that work as development tools to facilitate content creation and integration of the art material with the game or application executables. flyEditor.exe is one of the most important and useful tools of these. It can create a new .fly file or open an existing one, and serves for editing global or engine variables, loading plugins, creating objects, activating them, editing parameters in any object, be it active or not, and much more. flyEditor.exe also features a rendering window where the simulation of the currently open .fly file is executed and viewed, making it possible to modify parameters and see the results on-the-fly.

flyEditor.exe has a window menu with various options, and its window is divided in three parts: the tree view, the list view and the render view. All these items are discussed in the following sessions:

The Window Menu

flyEditor.exe features a complete window menu, with useful options like creating or loading a .fly file, copying and pasting of objects and parameter values, simulation options, and much more. Most commands, specially in the Edit submenu, are also available by right-clicking the desired object.

The File submenu has several items: New creates a new .fly file from scratch; Open opens a previously created .fly file; Save and Save As save the currently loaded file; Revert to Saved throws away all modifications made to the currently loaded .fly file and re-opens the previously saved version of the file; Restart restarts the simulation; Save Fly3D.ini saves the initialization variable values on the 'Fly3D.ini' file; a list of recently loaded .fly files for quick opening; and the Exit option, which closes the application.

The Edit submenu includes menu items for dealing with objects, like the Cut, Copy and Paste commands; the Duplicate item makes a copy of the currently selected object in the tree view, and pastes it in the same hierarchical position of the original object; Insert can be used in two ways: to insert a new plugin, .f3d object or texture (if any of these three tree view root items is selected) or to create a new instance of a class (if a class inside a plugin is selected instead); Delete/Clear can be used to delete all instances of the selected class or to remove the selected plugin from the scene; Rename can be used to rename any class instance in the simulation.

The View submenu has 4 options related to viewing and rendering: Object Only focuses the camera on the selected object and draws it as seen in the simulation; Simulate turns the simulation on/off; Toolbar and Status Bar toggle the respective controls on/off.

And finally, the Help submenu has only one item: About flyEditor, which displays information concerning the development of the application and copyright issues.

The Toolbar Menu

Right below the window menu is a toolbar with useful buttons for quick access to some functions of flyEditor.exe. Most of them are also accessible through the window menu.

The complete list of buttons and the action each of them perform, in the correspponding order that they are placed in the toolbar, from left to right, can be found below:

  • the New, Open and Save buttons correspond to the options with the same name found in the File submenu of the window menu;
  • SR (Save and Restart) saves the current .fly file and reloads it right after, restarting the simulation;
  • RV (Revert to Saved) matches the option with the same name in the File submenu of the window menu;
  • Cut, Copy, Paste and Duplicate work the same way as their counterparts in the Edit submenu of the window menu;
  • The Arrow Up and Arrow Down buttons are used to change the order in which the classes are displayed below the corresponding DLL item in the tree view;
  • The Object Only and Simulate buttons are switches, in the same way as the options with the same name in the View submenu of the window menu;
  • The About button displays information about the development of flyEditor.exe and its copyright.

The Tree View

The leftmost part of the flyEditor.exe window is the tree view. A tree view is a Windows control that displays items organized as a hierarchy. flyEditor.exe's tree view displays several basic items (the "root-items"), that can be expanded to show subitems. Upon selecting an item on the tree view, the list view beside it is updated, displaying a list of parameters corresponding to the selected item. Each of the root items, as well as the subitems appliable to them, are discussed below.

The Global root item represents Fly3D global and flyEngine's variables. When this item is selected, all the variables and their values are displayed in the list view, making it possible to modify them.

The Plugins root item represents all the plugin DLLs, classes and instances that exist in the currently loaded .fly file. After selecting the Plugins root item, it is possible to insert a new plugin DLL to the simulation. Any DLLs present in the current file are displayed one level inside the Plugins root item. One level further are the classes implemented in each plugin DLL. These can be selected for creating a new instance of the class, as a stock object. The existing instances of a given class are represented by subitems below the class item in the hierarchy, and can be selected for modifying their parameters, which appear in the list view.

The Objects root item represents all the .f3d and .k3d objects existing in the currently loaded .fly file. Upon selecting it, all the .f3d and .k3d objects in the file are displayed in the list view, together with some parameters for them, like name, number of animations, etc.

The Sounds root item represents all the sound files existing in the currently loaded .fly file. Upon selecting it, all the sound files are displayed in the list view, together with some parameters for them, like name, length, format, etc.

The Textures root item represents the whole set of textures applied to the faces in the currently loaded level. Upon selecting it, all the textures and lightmaps are displayed in the list view, together with some of their parameters, like name, size, etc.

The Shaders root item represents all shaders applied to the faces in the currently loaded level. Upon selecting it, all the shaders are displayed in the list view, together with some of their parameters, like name, number of passes, etc.

The List View

The central part of the flyEditor.exe window houses the list-view. A list view is a Windows control that displays a table, where the lines represent items of data and the columns represent fields in which items' parameters are represented.

Depending on the item selected in the tree view, the list view can display parameters relative to a given instance of a class, as an active or stock object; a list of textures, shaders, sounds or .f3d and .k3d mesh objects; or a list of global variables.

When displaying a list of object parameters, the list view enumerates them and displays their values; by double-clicking the left mouse button on a parameter, the corresponding value edition dialog box is displayed, and the parameter's value can be changed.

When displaying a list of textures, shaders or .f3d and .k3d files, selecting an item displays the corresponding texture or shader or mesh in the render view to the right. These entities can be rotated by dragging the left mouse button in the render view; zooming can also be performed, by dragging the right mouse button on the render view.

When displaying a list of sounds, double-clicking the left mouse button on a sound plays the corresponding .wav file.

The Stock/Active Objects Menu

Finally, on the upper rightmost part of flyEditor.exe is located the Stock and Active Objects menu. Here, it is possible to create active objects from stock objects, inserting them into the simulation. Also, this menu allows the selection of active objects created from a given stock object. It is also possible to destroy an active object, removing it from the simulation immediately.

When an instance of a class is selected in the tree-view, the list view displays a list of parameters corresponding to the respective stock object. If the Active Objects check box in the Stock and Active Objects menu is checked, then the list view will display the list of parameters of the first active instance of the object. With the arrow buttons, the current active object can be changed. The Activate button creates a new active object as a clone of the selected stock object; the Destroy button removes the currently selected active object from the simulation.
